## 1. 프로젝트 개요

### 1.1 프로젝트 목적

"할 게임이 없다"고 느끼는 게이머들을 위한 **개인화 게임 추천 플랫폼** 개발

### 1.2 핵심 가치

| 가치 | 설명 |
|------|------|
| **Connectivity** | Steam 계정 연동으로 게임 라이브러리 자동 분석 |
| **Personalization** | 플레이 타임 + 평가 데이터 기반 정교한 추천 |
| **Hybrid Intelligence** | 협업 필터링 + 컨텐츠 기반의 하이브리드 추천 |
| **Optimization** | DB 캐싱으로 RAWG API 의존도 최소화 (98% 속도 향상) |
| **AI Assistance** | GPT-5 기반 AI 큐레이터와 대화형 추천 |

### 1.3 기술 스택

```
Backend:  Python 3.12 + Django 5.2
Frontend: Django Templates + Vue.js 3.0
Database: SQLite (개발) / PostgreSQL (운영)
AI/ML:    GPT-5 Nano, Gemini 2.0 Flash Lite
APIs:     Steam Web API, RAWG API, CheapShark API
Libraries: scipy, sklearn, pandas, numpy
```

## 4. 비기능 요구사항

### 4.1 성능

| 항목 | 요구사항 |
|------|---------|
| 페이지 로딩 | 메인 페이지 2초 이내 (캐시 적용 시 0.15초) |
| API 응답 | 추천 API 1초 이내 |
| 동시 접속 | 100명 동시 접속 지원 |
| 배치 작업 | 게임 유사도 계산 5분 이내 (게임 1,000개 기준) |
| **Top-K 쿼리** | **similarity_rank 인덱스로 0.01초 이내** |

### 4.2 보안

| 항목 | 요구사항 |
|------|---------|
| 비밀번호 | Django 기본 해시 (PBKDF2) |
| Steam 연동 | OpenID 2.0 표준 준수 |
| CSRF | Django CSRF 토큰 적용 |
| API 키 | 환경 변수로 관리 (.env) |

### 4.3 확장성

| 항목 | 요구사항 |
|------|---------|
| 유저 확장 | 유저 기반이 아닌 게임 기반 유사도로 O(G²) 유지 |
| DB 마이그레이션 | SQLite → PostgreSQL 전환 용이 |
| 캐시 레이어 | Redis 도입 가능한 구조 |
| **저장 공간** | **정규화 저장으로 50% 절약** |
| **가중치 튜닝** | **SIMILARITY_WEIGHTS 설정 파일로 조정 가능** |

## 7. 추천 알고리즘 (하이브리드)

### 7.1 핵심 설계 원칙

```
❌ 잘못된 접근:
   "게임을 장르 벡터로만 비교한다" → 실무에서 거의 안 씀

⭕ 올바른 접근 (표준):
   게임 벡터 = 여러 신호의 결합
   - 협업 필터링 임베딩 (유저 평가 기반) ← 메인
   - 장르/태그 벡터 ← 보조
   - 메타 점수 ← 보조
```

### 7.2 알고리즘 우선순위

```
1순위: Steam 라이브러리 기반 추천 (Steam 연동 시)
       └─ 상위 플레이타임 게임 장르 분석 → RAWG API 추천

2순위: 하이브리드 추천 (평가 데이터 있을 시)
       └─ 협업필터링(70%) + 장르(20%) + 메타점수(10%)

3순위: Content-Based Filtering
       └─ 좋아한 게임의 장르 분석 → 같은 장르 게임 추천

4순위: 인기 게임 폴백
       └─ JSON 데이터셋에서 리뷰 많은 순 추천
```

### 7.3 하이브리드 유사도 수식

```
┌─────────────────────────────────────────────────────────────────┐
│                    하이브리드 유사도 계산                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  final_similarity = 0.70 × collaborative_similarity             │
│                   + 0.20 × genre_similarity                     │
│                   + 0.10 × metacritic_similarity                │
│                                                                 │
│  where:                                                         │
│    - collaborative: GameSimilarity 테이블 (사전 계산)           │
│    - genre: Jaccard(tags_a, tags_b) = |A∩B| / |A∪B|            │
│    - metacritic: 1 - |score_a - score_b| / 30                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 7.4 가중 추천 점수 계산

```
┌─────────────────────────────────────────────────────────────────┐
│                    유저별 추천 점수 계산                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  weighted_score(game_b) =                                       │
│                                                                 │
│      Σ(hybrid_similarity(game_a, game_b) × normalized_rating)   │
│      ─────────────────────────────────────────────────────────  │
│                     Σ(normalized_rating)                        │
│                                                                 │
│  where:                                                         │
│    - game_a: 유저가 좋아한 게임들                                │
│    - game_b: 추천 후보 게임                                      │
│    - normalized_rating: 3.5→0.7, 5→1.0 (정규화된 평점)          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 7.5 평점 정규화

| 원본 점수 | 정규화 | 설명 |
|-----------|--------|------|
| -1 | -1.0 | 역따봉 |
| 0 | 0.0 | 스킵 (실제로는 필터링) |
| 3.5 | 0.7 | 따봉 |
| 5 | 1.0 | 쌍따봉 |

### 7.6 배치 계산 (게임 유사도)

```bash
# 배치 명령어
python manage.py calculate_game_similarity

# 옵션
--min-ratings 3    # 최소 평가 수 (기본: 3)
--top-k 50         # 게임당 저장할 유사 게임 수 (기본: 50)
--min-similarity 0.1  # 최소 유사도 (기본: 0.1)
--dry-run          # 테스트 모드 (저장 안함)
```

### 7.7 정규화 저장 규칙 (필수!)

```
⚠️ game_a_id < game_b_id 규칙

예시:
  Game(id=5) ↔ Game(id=12): similarity=0.85

저장 방식:
  ✅ game_a_id=5, game_b_id=12, score=0.85
  ❌ game_a_id=12, game_b_id=5, score=0.85  (저장 안함!)

효과:
  - 저장 공간 50% 절약
  - unique index 의미 명확화
  - 쿼리 단순화
```

