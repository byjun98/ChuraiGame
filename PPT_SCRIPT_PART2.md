## 슬라이드 14: API 관계도

### 📌 화면 구성
| API | 용도 | 호출 빈도 |
|-----|------|----------|
| Steam Web API | 로그인, 라이브러리 | 로그인 시 1회 |
| Steam OpenID | OAuth 인증 | 로그인 시 1회 |
| RAWG API | 게임 상세 정보 | 캐싱 후 6시간마다 |
| CheapShark | 할인 정보 | 배치 업데이트 |
| GMS (GPT) | AI 챗봇 | 메시지당 1회 |
| GMS (Gemini) | 번역 | 번역 요청 시 |

### 🎙️ 대사
> "API 관계를 자세히 보면,  
> 
> **Steam API**는 사용자 로그인과 게임 목록을 가져오는 데 사용합니다.  
> 처음 한 번 가져온 뒤 **DB에 캐싱**해서 다음엔 바로 불러옵니다.  
> 
> **RAWG API**는 게임 정보를 가져오는데, 매번 호출하면 느려서  
> **6시간마다 캐시를 갱신**하도록 최적화했습니다.  
> 원래 9초 걸리던 게 **0.15초로 98% 단축**됐습니다."

- 주요 테이블 설명 박스

### 🎙️ 대사
> "ERD입니다. 주요 테이블 설명드리면,  
> 
> **User**: 사용자 정보, Steam ID 연동 여부  
> **Game**: 게임 기본 정보, Steam App ID, RAWG ID  
> **GameRating**: 사용자의 게임 평가 (-1, 0, 3.5, 5점)  
> **GameSimilarity**: **미리 계산된 게임 간 유사도**  
> 
> 특히 GameSimilarity는 **정규화**를 적용해서  
> `game_a_id < game_b_id` 규칙으로 **저장 공간을 50% 절약**했습니다.  
> 
> (A,B)와 (B,A)가 같은 유사도니까, 하나만 저장하는 거죠."

## 슬라이드 16: 유사도 알고리즘 설명

### 📌 화면 구성
- Item-Based CF 시각화
- 코사인 유사도 공식
- 하이브리드 가중치 차트

### 🎙️ 대사
> "추천 알고리즘은 **하이브리드 방식**입니다.  
> 
> **1. 협업 필터링 (70%)**  
> '이 게임 좋아한 사람들이 저 게임도 좋아했다'  
> 코사인 유사도로 게임 간 유사도 계산  
> 
> **2. 장르 유사도 (20%)**  
> Jaccard Index로 태그 겹치는 정도 계산  
> 교집합 / 합집합  
> 
> **3. 메타크리틱 점수 (10%)**  
> 비슷한 평점대 게임끼리 묶기  
> 
> 이 3개를 가중합해서 최종 점수를 만듭니다."

## 슬라이드 17: AI 챗봇 설명

### 📌 화면 구성
- 챗봇 UI 스크린샷
- 시스템 프롬프트 일부

### 🎙️ 대사
> "AI 게임 큐레이터입니다. **GPT-5 Nano** 기반이에요.  
> 
> 핵심은 **Context-Aware 추천**입니다.  
> 단순히 '게임 추천해줘'가 아니라,  
> **사용자의 Steam 라이브러리와 평가 데이터**를 프롬프트에 주입합니다.  
> 
> 예를 들어:  
> '이 유저는 엘든링 100시간, 다크소울 50시간 플레이했다'  
> '좋아한 게임: 젤다, 위쳐3'  
> '싫어한 게임: 피파, 스포츠 류'  
> 
> 이 정보를 보고 AI가 **개인화된 추천**을 해줍니다."
